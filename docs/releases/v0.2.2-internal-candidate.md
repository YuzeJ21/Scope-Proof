## Reproducible verification commands

The candidate was refreshed on 2026-07-20 from local branch `codex/v022-rc-audit`. Its product
source remained `22594c364ec9c165aca264e186a9af325ea2bcab`; subsequent local-only changes contained

The following commands passed from the isolated worktree:

```bash

  --cov-fail-under=95

```

Observed results were:

- 1,004 passed and one intentional skip in the complete suite; combined coverage was 95.84%.
- 356 export, schema, and presentation checks; 312 evidence-integrity checks; and 197 workbench,
  launcher, accessibility-contract, and repository-contract checks passed.

  `779740ebbbe8f9e4155441c5437451384f8b6f117a2b6ee09036ce57586b9d5d`.
- The source distribution contained 475 entries. Its SHA-256 is intentionally recorded only in
  the external verification handoff: this document is itself included in the source distribution,
  so embedding the distribution's own digest here would change the artifact and invalidate that
  digest. The build command above plus `shasum -a 256` produces the final external value.
- Neither distribution contained review storage, coverage files, virtual environments, Git
  metadata, worktrees, caches, or bytecode.
- A clean Python 3.12 environment installed 48 compatible packages. The installed CLI reported
  version 0.2.2, both benchmarks passed, and the installed workbench health endpoint returned
  `ok`.
- A four-criterion installed review returned `blocked` at exact head `head-demo-002`. Markdown,
  JSON, CSV, and HTML exports were non-empty and preserved that head and criterion identities; the

- The copyable owner-rehearsal runbook completed in approximately 1.37 seconds of aggregate command
  runtime and remained Stage-1-ineligible. This is automated engineering timing, not participant
  completion-time evidence.

On macOS, the owner-rehearsal runbook's `realpath` step is required because `/tmp` is a symbolic
link to `/private/tmp`; the storage guard correctly rejected an unresolved `/tmp/...` ancestor and
accepted the documented real path. Only Python 3.12 was locally available, so this refresh makes no
