# ScopeProof v0.1.17 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish one protected, checksum-verifiable v0.1.17 release containing the 13 verified first-use, evidence-integrity, Action-contract, and local-storage improvements merged since v0.1.16.

**Architecture:** Finalize release identity and README guidance in one commit, then pin the copyable Action example to that immutable release-tree commit in a second commit. After protected merge, build from the exact merge SHA, publish exactly one tag and release, then independently redownload and verify both assets.

**Tech Stack:** Python 3.12, Pydantic 2, Hatchling, pytest, Ruff, Streamlit, GitHub Actions, GitHub CLI.

## Global constraints

- Version, wheel metadata, module, CLI, web launcher, and new-review provenance are exactly `0.1.17`.
- The copyable Action pin is a full 40-character commit containing the exact release package tree.
- The release has exactly one wheel and one matching checksum asset.
- Controlled tests, demo, benchmark, and health checks are not external PR runtime evidence or customer validation.
- Do not create issue comments, dogfood updates, reviewer requests, intermediate releases, or fork tests.
- No product schema, gate, lifecycle, record-version, permission, or dependency change belongs in this preparation PR.
