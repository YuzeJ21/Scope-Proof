# ScopeProof v0.1.20 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish one protected, checksum-verifiable v0.1.20 release containing the 12 verified
operator-recovery, draft-persistence, runtime-record, and decision-attribution improvements merged
since v0.1.19.

**Architecture:** Finalize release identity and README guidance in one commit, then pin the
copyable Action example to that immutable release-tree commit in a second commit. After protected
merge, build from the exact merge SHA, publish exactly one tag/release, and independently
redownload and verify both assets.

**Tech Stack:** Python 3.12, Hatchling, pytest, Ruff, GitHub Actions, GitHub Releases, Streamlit
loopback health.

## Global Constraints

- Version, wheel metadata, module, CLI, web launcher, and new-review provenance are exactly
  `0.1.20`.
- The Action pin is a full 40-character commit containing the exact v0.1.20 package tree.
- The release has exactly one wheel and one matching checksum asset.
- Controlled tests, benchmark, demo, and health checks are not external PR runtime evidence,
  customer validation, or proof of correctness.
- No issue comments, dogfood updates, manually sent emails, intermediate releases, or forks.
- No schema, gate, lifecycle, record-version, permission, dependency, or product behavior change
  belongs in the preparation PR.
