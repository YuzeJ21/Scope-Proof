# ScopeProof v0.1.15 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish one protected, checksum-verifiable v0.1.15 release containing the accumulated
first-use and auditability improvements since v0.1.14.

**Architecture:** Prepare release identity and documentation in one protected branch, then pin the
copyable Action example to the immutable first implementation commit so the source pin contains the
exact v0.1.15 package tree without a circular self-reference. After protected merge, build from the
exact merge commit, publish one tag and release, and independently redownload and reverify both
public assets.

**Tech Stack:** Python 3.11+, Hatchling, pytest, Ruff, Streamlit, GitHub Actions, GitHub CLI.

## Global Constraints

- `scopeproof_core/version.py` remains the only checked-in version value.
- Release version, wheel metadata, module version, console versions, and new-review provenance are
  exactly `0.1.15`.
- The copyable Action example uses a full 40-character immutable commit SHA, never a branch or tag.
- The release has exactly one wheel and one matching `.sha256` asset.
- Local demo, tests, benchmark, and health checks are controlled product verification, not external
  PR runtime evidence or customer validation.
- Do not create issue comments, dogfood updates, reviewer requests, intermediate releases, or fork
  tests.
- Evidence, gate, persistence, runtime-evidence, resolution, and final-acceptance behavior remain
  unchanged.
- No billing, paid API, LLM API, organization, second account, private repository, license,
  untrusted-code execution, generic review, security scanning, or auto-fix capability is added.
