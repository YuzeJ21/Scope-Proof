# ScopeProof v0.1.18 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development to implement tasks with independent
> review.

**Goal:** Publish one protected, checksum-verifiable v0.1.18 release containing
the 29 verified trust-boundary, lineage, runtime-guidance, rendering, Action,
and local-deletion improvements merged since v0.1.17.

**Architecture:** Finalize release identity and README guidance in one commit,
then pin the copyable Action example to that immutable release-tree commit in a
second commit. After protected merge, build from the exact merge SHA, publish
exactly one tag/release, and independently redownload and verify both assets.

## Global constraints

- Version, wheel metadata, module, CLI, web launcher, and new-review provenance
  are exactly `0.1.18`.
- The Action pin is a full 40-character commit containing the exact release
  package tree.
- The release has exactly one wheel and one matching checksum asset.
- Controlled tests, benchmark, demo, and health checks are not external PR
  runtime evidence or customer validation.
- No issue comments, dogfood updates, emails, intermediate releases, or forks.
- No schema, gate, lifecycle, record-version, permission, dependency, or product
  behavior change belongs in the preparation PR.

## Task 1: Finalize release identity and README contract

**Files:** `tests/test_repository_contracts.py`,
`scopeproof_core/version.py`, `README.md`.

1. Change repository contracts first from v0.1.17/`0.1.18.dev0` to
   v0.1.18/`0.1.18`.
2. Run the three exact release-install/version contract tests and require RED.
3. Set the version source to `0.1.18`; replace only release wheel/checksum paths
   and commands in README.
4. Repeat focused tests and require GREEN; probe new-review identity.
5. Commit as `Prepare v0.1.18 release` and record the full SHA as
   `RELEASE_TREE_COMMIT`.

## Task 2: Refresh the immutable Action source pin

**Files:** `tests/github_action/test_workflow_files.py`,
`examples/github-actions/scopeproof.yml`.

1. Require the release-tree SHA to be 40 characters and its version source to
   contain `0.1.18`.
2. Change the test's expected source SHA first and require the focused test RED.
3. Replace only the Git source SHA in the example; run the full workflow-file
   suite and require GREEN.
4. Commit as `Pin Action example to v0.1.18 tree`.

## Task 3: Verify the release candidate

1. Run Ruff, full pytest, `scopeproof benchmark`, and `git diff --check`.
2. Build one candidate wheel from `git archive HEAD` under `/tmp` and require
   exactly `scopeproof-0.1.18-py3-none-any.whl`.
3. Install into a fresh environment outside the checkout; require `pip check`
   and exact distribution/module/CLI/web/new-review identity.
4. Run the installed benchmark and exact loopback `ok` workbench health smoke
   with clean shutdown and no traceback.
5. Independently review the whole branch and require no unfixed Critical or
   Important finding.

## Task 4: Integrate through protected main

1. Push `codex/release-v0-1-18` and clean-install the public full-SHA package.
2. Open one ready PR describing the 29-merge batch, exact candidate results,
   evidence limits, and no-cost/no-fork boundaries.
3. Require both `verify` checks and CodeQL Python/Actions success; merge only
   from a clean merge state.
4. Fast-forward local `main`; require clean `HEAD == origin/main`; rerun local
   gates and require exact merged-main CI/CodeQL success.

## Task 5: Publish and independently verify v0.1.18

1. Archive exact merged main, build one wheel, generate its `.sha256`, and
   verify it locally.
2. Fresh-install and repeat dependency, identity, benchmark, and loopback
   health verification; record SHA and digest.
3. Create tag/release v0.1.18 at the exact merge SHA with only wheel and checksum
   assets and bounded release notes. Do not post an issue update or email.
4. Redownload both public assets into a different directory; require exactly two
   files, matching digest/checksum, clean install, identity, benchmark, and
   workbench health outside the checkout.
5. Reconcile tag/release target, assets, local/remote main, open PRs, exact-main
   CI/CodeQL, alerts, Actions pins, and genuine external evidence before the next
   loop.
