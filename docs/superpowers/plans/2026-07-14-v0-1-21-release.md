# ScopeProof v0.1.21 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this
> plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish one protected, checksum-verifiable v0.1.21 release that makes the already merged
and documented `scopeproof list` command available through the verified public wheel.

**Architecture:** Finalize release identity and README guidance in one commit, then pin the
copyable Action example to that immutable release-tree commit in a second commit. After protected
merge, build from the exact merge SHA, publish exactly one tag/release, and independently
redownload and verify both assets and the installed list command.

**Tech Stack:** Python 3.12, Hatchling, pytest, Ruff, GitHub Actions, GitHub Releases, Streamlit
loopback health.

## Global Constraints

- Version, wheel metadata, module, CLI, web launcher, and new-review provenance are exactly
  `0.1.21`.
- The Action pin is a full 40-character commit containing the exact v0.1.21 package tree.
- The release has exactly one wheel and one matching checksum asset.
- Controlled tests, benchmark, demo, and health checks are not external PR runtime evidence,
  customer validation, or proof of correctness.
- No issue comments, dogfood updates, labels, reviewers, manually sent emails, intermediate
  releases, or forks.
- No schema, gate, lifecycle, record-version, permission, dependency, or product behavior change
  belongs in the preparation PR.

### Task 3: Verify the release candidate

- [ ] **Step 1: Run repository-wide gates**

Require Ruff, the full pytest suite, the deterministic benchmark, `pip check`, and
`git diff --check origin/main...HEAD` to pass. The benchmark must report zero mismatches, zero
must-have False Ready outcomes, zero false blockers, zero evidence-link errors, and zero
unexecuted categories.

- [ ] **Step 2: Build from an archive**

Export `git archive HEAD` to a fresh `/tmp` directory and build exactly one
`scopeproof-0.1.21-py3-none-any.whl`.

- [ ] **Step 3: Verify a clean install**

Install the wheel with dependencies in a fresh environment outside the checkout and require
`pip check`. Require distribution, module, CLI, web launcher, and new-review provenance identity
to agree on `0.1.21`.

- [ ] **Step 4: Verify installed behavior**

Run installed `scopeproof list` against fresh stores and require successful empty output, sorted
valid review IDs, and corrupt-entry isolation. Run the installed benchmark and require the same
zero-error metrics. Start installed `scopeproof-web` on loopback with a fresh `HOME`, require
`/_stcore/health` to return `ok`, then stop it without traceback.

- [ ] **Step 5: Review the candidate diff**

Require the base-to-head diff to contain only design, plan, release identity, README guidance,
Action pin, and their tests. Require no unfixed Critical or Important finding.

Require tag and release target equality, exactly two release assets, clean local/remote main,
