# Engineering Archive Index

The `specs/` and `plans/` directories contain historical engineering records retained for design
and implementation provenance. They explain decisions and intended work at the time each record
was written.

These records are not current product status, not runtime evidence, not adoption evidence, and
not a sequential user manual. A completed plan also does not prove that a feature works in a current
checkout or published release.

Use the repository's current sources of truth instead:

- [`README.md`](../../README.md) for supported workflows and product boundaries.
- [`ROADMAP.md`](../../ROADMAP.md) for evidence-gated stage status and waiting conditions.
- [`CHANGELOG.md`](../../CHANGELOG.md) and GitHub Releases for published change history.
- Current tests and pinned GitHub Actions workflows for implementation and verification evidence.

Treat any claim in an archived record as historical unless it is independently supported by those
current sources.
